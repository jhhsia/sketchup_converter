Creates a new image object from an image file on disk.

Parameters
[out]imageThe image object.
[in]boundsThe bounding box for the image.
[in]file_pathThe file path to the image on disk.
Returns
  • SU_ERROR_NONE on success
  • SU_ERROR_NULL_POINTER_OUTPUT if image is NULL
  • SU_ERROR_OVERWRITE_VALID if *image already refers to a valid object
  • SU_ERROR_NULL_POINTER_INPUT if bounds is NULL
  • SU_ERROR_NULL_POINTER_INPUT if file_path is NULL
  • SU_ERROR_OUT_OF_RANGE if bounds has a width or height of zero
  • SU_ERROR_SERIALIZATION if there was an error reading the image file or allocating the memory for the image
  • SU_ERROR_NO_DATA if the image file could not be found

LO_RESULT LOImageSetClipMask ( LOImageRef  model,
LOEntityRef  clip_mask 
)

Sets the clip mask of the image. A clip mask defines a region of the entity that is visible. This allows you to crop with arbitrary shapes. This operation will replace any clip mask that is already assigned to this image. The entity being used must not be already part of a document or group. The clip mask entity must be either a rectangle, ellipse or a path.

Since
LayOut 2017, API 2.0
Parameters
[in]modelThe image object.
[in]clip_maskThe new clip mask for the image.
Returns
  • SU_ERROR_NONE on success
  • SU_ERROR_INVALID_INPUT if image does not refer to a valid object
  • SU_ERROR_INVALID_INPUT if clip_mask does not refer to a valid object
  • SU_ERROR_GENERIC if the clip_mask is already in a document.
  • SU_ERROR_GENERIC if the clip_mask is not a rectangle, ellipse, or path.
  • SU_ERROR_LAYER_LOCKED if image is on a locked layer.
  • SU_ERROR_ENTITY_LOCKED if image is locked
